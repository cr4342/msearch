# 实现计划

## 开发策略

按照以下顺序进行增量开发：
1. **阶段一：基础架构** - 搭建项目结构、配置管理、日志系统
2. **阶段二：文件扫描与SQLite** - 实现文件监控和基础数据存储
3. **阶段三：以文搜图** - 集成Infinity和CLIP，实现图像向量化和检索
4. **阶段四：API与验证** - 搭建FastAPI服务，验证核心功能
5. **阶段五：UI集成** - 引入PySide6，实现基础交互界面
6. **阶段六：多模态支持** - 加入视频和音频处理，完善多模态检索
7. **阶段七：后端优化** - 优化核心功能，提升性能和稳定性
8. **阶段八：部署与监控** - 部署到Linux服务器，配置监控和自动化部署

## 关键设计更新

- **硬件分析**：由安装脚本在安装阶段执行，不再使用HardwareDetector组件
- **调度器配置**：支持通过配置文件设置调度检查间隔
- **指数退避重试**：为失败任务添加指数退避重试策略，特别是向量化等依赖外部服务的操作
- **健康检查端点**：为各组件添加健康检查接口，方便监控和调试
- **统一错误码**：定义统一的错误码体系，便于问题定位和监控
- **工作流设计**：调度器协调各组件完成文件处理与向量化流程
- **安装脚本**：跨平台支持，负责系统初始化、硬件分析和配置生成
- **资源下载**：所有资源下载到temp目录，不再使用offline目录

**关键目录说明**:

1. **src/** - 源代码按层次清晰分离
   - api/: API服务层，处理HTTP请求
   - business/: 业务逻辑层，核心功能实现
   - processors/: 媒体处理器，专门处理不同类型文件
   - storage/: 数据存储层，抽象化数据访问
   - core/: 核心组件，提供基础功能
   - gui/: 用户界面，PySide6实现

2. **config/** - 配置文件集中管理
   - 支持多环境配置（dev/prod）
   - 模型配置独立管理
   - 告警规则可配置
   - 调度器检查间隔可配置

3. **data/** - 数据文件分类存储
   - models/: AI模型文件（按模型类型组织）
   - database/: SQLite数据库
   - qdrant/: 向量数据库
   - temp/: 临时文件（可自动清理）
   - backups/: 备份文件
   - checkpoints/: 任务断点

4. **tests/** - 测试按类型组织
   - unit/: 单元测试（60%）
   - integration/: 集成测试（30%）
   - e2e/: 端到端测试（10%）
   - performance/: 性能测试
   - accuracy/: 精度测试
   - testdata/: 测试数据

5. **logs/** - 日志文件（项目根目录）
   - msearch.log: 主日志文件
   - error.log: 错误日志
   - performance.log: 性能日志
   - 自动轮转和归档

6. **scripts/** - 自动化脚本
   - 安装脚本（在线/离线/兼容）: Python编写，跨平台支持
   - 服务管理脚本
   - 打包和分发脚本
   - 维护和备份脚本

7. **特殊目录**:
   - deploy_test/: 部署测试环境（不纳入Git）
   - migrations/: 数据库版本迁移脚本

# 安全性设计文档

## 1. 概述

MSearch 是一款单机可运行的跨平台多模态桌面检索软件，专为视频剪辑师设计。系统采用多层次安全防护机制，确保数据安全和系统稳定运行。

### 1.1 设计目标

- **数据安全**: 保护用户文件和索引数据不被未授权访问
- **隐私保护**: 确保用户数据本地存储，不依赖互联网
- **系统稳定**: 防止恶意输入和异常情况导致系统崩溃
- **可审计性**: 记录关键操作，便于问题追踪和安全审计

### 1.2 安全原则

- **最小权限原则**: 系统仅请求必要的文件访问权限
- **纵深防御**: 多层次安全防护，单点失效不影响整体安全
- **默认安全**: 默认配置采用安全设置，用户需显式配置才能降低安全级别
- **透明性**: 安全机制对用户透明，不影响正常使用体验

## 2. 数据安全

### 2.1 文件访问控制

**访问权限管理**:
- 系统仅请求读取用户选择的目录和文件
- 不访问系统敏感目录（如系统配置、密钥存储等）
- 文件访问遵循操作系统权限控制

**访问范围限制**:
```
允许访问:
- 用户指定的媒体文件目录
- 用户指定的项目文件目录
- 系统生成的临时目录

禁止访问:
- 系统配置目录
- 其他用户的文件
- 网络共享资源（除非用户显式授权）
```

**文件扫描安全**:
- 文件扫描器仅扫描用户授权的目录
- 支持目录黑名单，排除敏感目录
- 扫描过程实时显示，用户可随时中断

### 2.2 数据加密

**存储加密**:
- 向量数据库（Milvus Lite）支持可选加密
- SQLite 数据库支持可选加密（SQLCipher）
- 配置文件敏感信息加密存储

**传输加密**:
- 本地进程间通信使用 Unix Domain Socket（Linux）或 Named Pipe（Windows）
- API 接口支持 HTTPS（如需远程访问）

**密钥管理**:
- 加密密钥由用户密码派生（PBKDF2）
- 密钥不存储在明文文件中
- 支持密钥轮换

### 2.3 数据备份与恢复

**自动备份**:
- 配置文件和数据库定期自动备份
- 备份文件存储在用户指定目录
- 支持备份版本管理

**恢复机制**:
- 系统启动时检查数据完整性
- 检测到损坏时自动从备份恢复
- 支持手动选择备份版本恢复

**备份安全**:
- 备份文件与原文件使用相同加密级别
- 备份目录访问权限与主目录一致

## 3. API 安全

### 3.1 认证与授权

**本地 API 访问**:
- 默认仅允许本地访问（127.0.0.1）
- 支持禁用 API 接口（纯桌面应用模式）
- API 访问需要认证令牌（如启用）

**认证机制**:
- 支持 API Key 认证
- 支持 JWT Token 认证
- Token 过期时间可配置

**授权模型**:
- 基于角色的访问控制（RBAC）
- 支持细粒度权限控制
- 默认角色为只读用户

### 3.2 输入验证

**文件路径验证**:
- 验证文件路径是否在允许范围内
- 防止路径遍历攻击（Path Traversal）
- 验证文件类型和大小限制

**查询参数验证**:
- 验证查询文本长度和内容
- 验证参数类型和范围
- 防止注入攻击

**文件上传验证**:
- 验证文件类型（MIME 类型和文件头）
- 限制文件大小
- 扫描恶意文件（可选）

### 3.3 输出过滤

**敏感信息过滤**:
- 日志中不记录完整文件路径
- 错误消息不暴露系统内部信息
- 响应数据不包含敏感元数据

**数据脱敏**:
- 文件路径脱敏（仅显示相对路径）
- 用户信息脱敏
- 错误堆栈脱敏（生产环境）

## 4. 日志安全

### 4.1 日志记录

**日志级别**:
- DEBUG: 详细调试信息（开发环境）
- INFO: 一般信息（生产环境）
- WARNING: 警告信息
- ERROR: 错误信息
- CRITICAL: 严重错误

**日志内容**:
- 记录关键操作（文件扫描、向量化、检索）
- 记录错误和异常
- 记录性能指标

### 4.2 敏感信息保护

**不记录的敏感信息**:
- 完整文件路径（记录相对路径或哈希值）
- 用户密码和密钥
- API Token 和认证信息
- 文件内容（仅记录元数据）

**日志脱敏规则**:
```python
# 文件路径脱敏
原始路径: /home/user/projects/video_project/clip_001.mp4
脱敏后:  .../video_project/clip_001.mp4

# 错误消息脱敏
原始错误: FileNotFoundError: [Errno 2] No such file or directory: '/home/user/secret/config.json'
脱敏后: FileNotFoundError: File not found: .../config.json
```

### 4.3 访问审计

**审计日志**:
- 记录 API 访问（IP、时间、操作）
- 记录文件访问（用户、时间、文件）
- 记录配置变更（用户、时间、变更内容）

**审计日志存储**:
- 审计日志单独存储
- 审计日志不可篡改
- 审计日志长期保留

**审计报告**:
- 支持生成审计报告
- 支持审计日志查询
- 支持异常行为检测

## 5. 本地运行安全

### 5.1 离线运行

**无互联网依赖**:
- 所有功能在本地运行
- 不需要互联网连接
- 不上传用户数据

**本地数据存储**:
- 所有数据存储在本地
- 不依赖云端服务
- 用户完全控制数据

### 5.2 进程隔离

**多进程架构**:
- 主进程：UI 和业务逻辑
- 工作进程：文件处理和向量化
- API 进程：API 服务（可选）

**进程间通信**:
- 使用安全的 IPC 机制
- 验证消息来源
- 限制消息大小

**资源限制**:
- 限制工作进程内存使用
- 限制工作进程 CPU 使用
- 防止资源耗尽攻击

### 5.3 依赖安全

**第三方库安全**:
- 使用官方发布的稳定版本
- 定期更新依赖库
- 检查已知漏洞

**模型文件安全**:
- 模型文件来自可信源
- 验证模型文件完整性
- 支持模型文件签名验证

## 6. 错误处理安全

### 6.1 异常处理

**异常捕获**:
- 捕获所有未处理的异常
- 记录详细的错误日志
- 返回友好的错误消息

**错误恢复**:
- 自动重试可恢复的错误
- 保存处理进度
- 支持从断点恢复

**错误隔离**:
- 单个文件处理错误不影响其他文件
- 单个任务错误不影响其他任务
- 错误不泄露系统内部信息

### 6.2 资源清理

**自动清理**:
- 自动清理临时文件
- 自动释放文件句柄
- 自动关闭数据库连接

**异常清理**:
- 异常发生时清理已分配资源
- 确保资源不泄漏
- 支持强制清理

## 7. 数据库安全

### 7.1 Milvus Lite 安全

**访问控制**:
- 仅允许本地访问
- 支持访问密码（可选）
- 支持连接白名单

**数据隔离**:
- 每个项目使用独立的 Collection
- Collection 访问权限隔离
- 支持数据分区

### 7.2 SQLite 安全

**文件权限**:
- 数据库文件权限设置为仅用户可读写
- 数据库文件位置隐蔽
- 支持数据库加密

**SQL 注入防护**:
- 使用参数化查询
- 验证输入参数
- 限制 SQL 语句复杂度

## 8. 安全配置

### 8.1 默认配置

**安全默认值**:
```python
# API 配置
api_enabled = False  # 默认禁用 API
api_bind_address = "127.0.0.1"
api_port = 8000
api_auth_required = True

# 数据加密
database_encryption = True  # 默认启用数据库加密
vector_db_encryption = True  # 默认启用向量数据库加密

# 日志配置
log_level = "INFO"
log_sensitive_data = False
log_audit_enabled = True

# 文件访问
allow_network_access = False
allow_system_dirs = False
```

### 8.2 安全策略

**密码策略**:
- 支持强密码要求
- 支持密码过期
- 支持密码历史

**访问策略**:
- 支持访问时间限制
- 支持访问频率限制
- 支持IP 白名单

**备份策略**:
- 支持自动备份
- 支持备份加密
- 支持备份保留策略

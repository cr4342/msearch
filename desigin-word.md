## 项目概述
该项目旨在开发一个基于内容理解的多模态检索系统，支持基于文本描述或图片检索目标文件夹中的相似图片和视频。系统的主要功能模块包括：
- **文件扫描**
- **向量化**
- **数据存储**
- **任务管理**
- **检索**

### 系统目标
- 允许用户通过文字描述或图片进行检索。
- 支持从文件夹中检索相似的图片和视频。
- 提供一个基于向量的检索机制，能够高效地处理文本、图片和视频数据。

---

## 技术栈

- **视频处理**：Towhee、ffmpeg
- **向量存储**：Milvus
- **特征提取模型**：Chinese-CLIP

---

## 设计思路

### 向量化
- **目标**：利用 Chinese-CLIP 大模型对待检索目标进行向量化，将生成的向量存储到向量数据库（Milvus）中。
  
### 检索流程
1. **用户输入**：用户可以通过文字描述或图片上传进行检索。
2. **向量化处理**：输入的数据（文字或图片）会经过 Chinese-CLIP 模型进行向量化处理。
3. **从数据库检索**：通过与数据库中存储的向量进行比对，查找相似的文件信息，并返回结果给用户。

## 关键服务功能逻辑

1. **任务管理系统**：通过 MySQL 开发一个全局任务管理系统，其中每个任务记录包括文件ID、MD5、文件类型、文件路径、向量化状态等信息。
   
2. **数据准备与任务队列**： 
   - 从本地扫描文件夹或通过 Nextcloud 的 API 获取文件时，创建任务记录，包括任务ID、文件MD5、原路径等信息。
   
3. **文件预处理服务**： 
   - 加载任务队列，将文件放入缓存路径，判断文件类型，更新文件类型。
   - 如果是视频类文件，调用 Towhee 生成每秒一帧的 GIF 预览图，并生成帧序列存入缓存。
   - 如果是图片文件，则生成统一尺寸的预览图（宽度为 512），节省后续资源占用。

4. **大模型服务**： 
   - 抽象成统一的向量生成接口，供主程序调用，用于对文字、图片、视频进行向量化。

5. **向量存储服务**： 
   - 部署 Milvus 服务端 Docker，通过客户端完成索引管理和检索。

## 系统架构设计

### 文件与任务处理模块

1. **init_sql.py**  
   - 初始化数据库表结构，创建任务管理系统的数据库（支持 MySQL、PostgreSQL 或 SQLite）。
   - `task` 表：任务记录包含任务ID、文件ID、文件名、来源、文件路径、MD5、任务类型、向量化状态等字段。

2. **task.py**  
   - 扫描本地或远程文件夹，生成任务队列。
   - 本地文件扫描时，根据文件MD5判断是否需要创建新任务。
   - 远程文件扫描时，使用 Nextcloud API 获取文件并缓存。

3. **utility.py**  
   - 提供文件扫描、远程文件下载、视频帧提取（生成 GIF 和帧序列）等文件预处理功能。

4. **vector.py**  
   - 读取任务队列，检查向量化状态，并对未处理的文件进行向量化处理。
   - 对图片使用 CN-Clip 模型生成向量，对视频则提取帧并生成相应的向量。

5. **main.py**  
   - 负责主流程的控制和任务执行，整合各模块功能，协调任务管理、文件处理、向量化与数据库交互。

6. **api.py**  
   - 将主程序封装为 API，方便后期通过 Web UI 进行交互。

### 任务管理模块

- **TaskManager**
  - `set_nextcloud_client(nextcloud_url, username, password)`：初始化 Nextcloud 客户端。
  - `add_remote_task(file_url)`：从 Nextcloud 下载文件，并创建任务记录。
  - `calc_md5(cache_path)`：计算文件的 MD5 哈希。
  - `get_task_type(cache_path)`：根据文件类型决定任务类型。
  - `create_task(**kwargs)`：在数据库中创建新任务。

### 数据库交互模块

- **Task 模型**
  - `id`：任务主键。
  - `file_id`：文件唯一标识符。
  - `file_name`：文件名。
  - `source`：任务来源（本地或远程）。
  - `file_path`：文件路径。
  - `md5`：文件 MD5 哈希值。
  - `created_at`：任务创建时间。
  - `cache_path`：文件缓存路径。
  - `cache_completed_at`：缓存完成时间。
  - `type`：任务类型（0表示文字，1表示图片，2表示视频等）。
  - `vectorized`：表示任务是否已经完成向量化。

### 文件处理模块

- **scan_files(folder_path)**：扫描指定目录中的文件。
- **file_type(file_path)**：根据文件扩展名确定文件类型。
- **pumping(file_path)**：从视频中提取帧并保存为图像。

### 向量化模块

- **vector_image(path, file_name)**：将图片向量化并返回特征向量。
- **process_video()**：处理视频文件，通过提取帧并生成向量。

### Milvus 集成模块

- **connect_milvus(host='localhost', port='19530')**：连接 Milvus 服务器。
- **create_collection(collection_name, dim)**：在 Milvus 中创建集合以存储向量。
- **insert_vectors(collection_name, id, file_paths, name, vectors, timestamps=None)**：将向量插入 Milvus 集合中。
